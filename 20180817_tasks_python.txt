

Задачи на питоне. Можно использовать Python 2.7.* или Python 3.*. Немного отличается синтакис.

1) Реализация процедуры создания полного пути каталогов.

Входные данные: 
  - строка содержащая полный путь каталога который надо создать. Например, r"C:\\catalog1\\catalog2\\catalog3".  Например, если catalog1 существует, а catalog2 и catalog3 - 
  то они должны быть созданы.
   
Возвращаемое значение:
  - результат выполнения функции в формате bool (True, если успешно создан путь каталогов или False при ошибке создания) 
    или в формате int, тут может быть больше вариантов, например: 0 - успешно, -1 - неправильный формат строки и так далее.
    
   Подготовить также модуль вызывающий данную процедуру.
   Например, при запуске этого модуля выводится сообщение: "Введите имя каталога", после этого сообщения пользователь вводит
   полный путь и нажимает Enter и получает сообщение либо об успешном создании каталога, либо об ошибке.

Дополнительные требования:
Не использовать функцию os.makedirs, использовать только os.makedir
   
2) Есть файл содержаший вывод какой либо программы. В этом файле могут присутствовать сообщения следующего формата:
  
  <начало строки или один и более пробелов после начала строки><Заголовок><один или более символов пробела>текст сообщения <символ конец строки> или конец файла
  
  Формат заголовка: MVFxxxxx<тип сообщения>. x - это цифра от 0 до 9.
  Тип сообщения: E - ошибка, W - предупреждение, I - информационное сообщение.
  Пример сообщений:
  
    MVF01095I  FOUND RECORD 
...любой текст    
     MVF02345W  ANY WARNING
...любой текст
    MVF22234E   ALL NOTHING WORK

Необходимо подготовить процедуру для поиска всех сообщений или сообщений определенного типа. 
   Входные данные: 
     - имя файла с данными программы; 
     - тип сообщений который ищем: 'E' (ошибки), 'W' (предупреждения), 'I' (инфо), 'A'( все сообщения, можно использовать как значение по умолчанию)
     
   выходные данные:
      - список сообщений, каждый элемент которого содержит следующую информацию:
          заголовок сообщения;
          тип сообщения;  
          смещение в байтах начала сообщения от начала файла; 
          текст сообщения.
          
          
  Например, для примера сверху выходные данные будут в таком виде (список словарей):
   [ {'msg': 'MVF01095I', 'type': 'I', 'offset': 102, 'text': 'FOUND RECORD'},
     {'msg': 'MVF02345W', 'type': 'W', 'offset': 232, 'text': 'ANY WARNING'},
     {'msg': 'MVF22234E', 'type': 'E', 'offset': 302, 'text': 'ALL NOTHING WORK'}]
     
  
   Подготовить также модуль вызывающий данную процедуру с простым консольным интерфейсом в виде "введите имя файла" и 
    "введите тип сообщений E-ошибки, W-предупреждения, I - информация, A - все сообщения".
    
    Прилагается файл, на котором можно проверить данное задание: "20180817_for_2_task.txt".
